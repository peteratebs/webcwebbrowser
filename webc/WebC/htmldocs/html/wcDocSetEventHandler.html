void wcDocSetEventHandler

(wcCtx* Ctx, wcEventCallback cb)

Assign an event handler to the body element of this document.

Documentation

Push the previous event handler for the body element on an internal stack and then sets a new handler for events targeted at the body. The previous event handler may be restored later by calling wcDocPopEventHandler.
Note: wcDocSetEventHandler pushes the previous handler onto an internal stack. You must call wcDocPopEventHandler at a later time if your application requires you to restore the previous handler when the current algorithm is completed.

Parameters:
wcCtx - * Ctx - The current wc context.
wcEventCallback - cb - Function to call when an event occurs. (see wcElSetEventHandler) wcDocSetEventHandler calls wcElSetEventHandler to attach the event handler to the documents body element.
Document event handlers are actually just element event handlers acting on the body element of a document but since the body element is the containing element for all other elements its event handlers provide some additional utility over other handlers for other elements.
  • Can be used to configure allocate and initialize application screen specific data prior to loading content into the page.
  • Can be used to configure release application screen specific data prior to release content from the page (loading new content or destroying the window).
  • Can be used to catch events like clicks that land on the document background.
  • Can be used to send events to the application itself and not necessarilly any of its elements.
  • Can be used to catch events that were processes by elements in the document but not canceled (bubbled).
The document body event handler is especially useful for these events -
  • HTML_EVENT_LOAD - Called when the document has been loaded with content by a call to wcDocSetUrl or wcDocClose or wcWinCreateEx or a Javascript equivalent. Note: Body elements of documents that were created using pure dynamic HTML are never sent an HTML_EVENT_LOAD event notification.
  • HTML_EVENT_UNLOAD - Called every time the current content is about to be cleared to prepare for loading another page. Application screen "OnExit" processing should be placed in HTML_EVENT_UNLOAD clauses.
  • HTML_EVENT_OBJECT_CREATED - Called every time the document is cleared . Application screen "OnEntry" processing can be placed in the HTML_EVENT_OBJECT_CREATED clause but remember that the document will be empty. If a url is being loaded or wcDocWriteHtml it is being used HTML_EVENT_LOAD will be triggered when the content is loaded.
Example: See ExDocEventHandling
Returns:
Nothing
